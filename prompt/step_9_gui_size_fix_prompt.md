## 背景

現状の GUI は機能追加により横方向に広がりすぎており、画面内に収まらないケースがあります。
特に以下のような問題が疑われます。

- ラベルと入力欄の横並びが多すぎる
- 長いパス表示欄が固定幅で広がっている
- タブ内に多数の controls を 1 行で詰め込みすぎている
- ダイアログの初期幅や minimum width が大きすぎる
- 画面サイズに応じたリサイズ制御が弱い
- スクロール領域がない、または一部タブでしか使っていない

今回の目的は、**GUI をノートPC など一般的な画面で無理なく収まるサイズにすること**です。

## 最重要修正方針

### 1. 横幅を優先して抑える
- GUI の最大幅を画面サイズ基準で制御する
- パス欄・ログ欄・表はダイアログ全体を押し広げない構造にする
- `QHBoxLayout` の使いすぎを見直し、必要なら `QFormLayout` や縦積みへ変更する
- 特に **ラベル + 長い入力欄 + ボタン** の 1 行構成を見直す

### 2. 縦方向はスクロールを許容する
- 横幅を無理に広げるより、縦方向スクロールを優先する
- 必要なら各タブを `QScrollArea` に入れる
- 主要な設定は見失いにくいよう、セクション見出しをつける

### 3. 長いパス表示に強くする
- OSV パス、作業フォルダ、YOLO モデルパスは、長くても GUI 全体を広げないようにする
- 入力欄は伸縮可能にしつつ、ダイアログの最大幅を超えて広がらないようにする
- 必要なら以下のいずれかを使う
  - elide 表示
  - tooltip で全パス表示
  - readOnly line edit + stretch 制御

### 4. 初期サイズ・最小サイズ・最大サイズを見直す
- 初期表示サイズは画面内に確実に収まる値にする
- minimum width / height を過剰に大きくしない
- 画面サイズを取得できる場合は、画面比率で適切なサイズを決める
- 例えば「画面幅の 80〜90% を上限」にする設計でよい

## レイアウト修正要件

## 1. ダイアログ全体
- GUI 初期表示時に画面外へはみ出さないこと
- 初期位置も極力画面内中央に近づけること
- 必要なら `resize()` / `setMaximumWidth()` / `setMaximumHeight()` / `adjustSize()` の使い方を見直すこと
- current Qt binding で安全に使える範囲で実装すること

## 2. タブ構成
既存タブ構成は維持してよいが、各タブ内レイアウトは見直してよい。

現在想定されるタブ:
- 基本設定
- 前処理
- マスク / 読込
- アライメント / 間引き
- ログ / 状態

要件:
- 各タブで横方向に広がりすぎない
- 必要なら各タブをスクロール可能にする
- ログタブだけ別途広い領域を持たせてもよいが、ダイアログ最大幅を超えないこと

## 3. 入力フォーム
以下のような横幅を食いやすい項目を重点的に見直す。

- 入力OSV
- 作業フォルダ
- YOLOモデルパス
- OpenCV 実行方式
- GPU 状態表示
- ログ保存先

推奨:
- 長い path 系はフォームの 1 行に詰めすぎない
- `入力欄 + 参照ボタン` を適切な stretch で構成する
- 状態表示は別行や group box 内へ逃がす

## 4. ボタン配置
- 処理ボタンを 1 行に並べすぎない
- 必要なら 2 行以上に分ける
- 重要ボタンのみ目立たせ、その他は縦積みや grid でもよい

---

## GUI 実装方針

### 推奨
- `QFormLayout` を優先して検討する
- `QGridLayout` を使う場合は列数を抑える
- 各タブの内容を `QScrollArea` に包める構造にする
- path 入力欄は stretch 付きで収める
- `QSizePolicy.Expanding` / `Preferred` / `MinimumExpanding` を適切に使う

### やってほしいこと
- 長すぎる fixed width を削除または緩和する
- minimum width が大きすぎる widget を見直す
- ログ欄はタブ内で広がっても、ダイアログ全体を無制限に広げないようにする
- 必要なら `splitter` の使用も可だが、まずは単純な scroll + layout 見直しを優先する
